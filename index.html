My Favourite Git Command

My Favourite Git Command is Git Commit.

Git commit is a command that records or the file permanently in the version history.

I will be showing you how to use the "git commit" in simple steps.

STEP 1:

  • Create a Github Account.
  • Install Git Bash(if not installed).

STEP 2:

  • Create a local repository.
  • Initialize an empty git repository by using the command "git init" in the root if your directory.

STEP 3:

  • Add a new file to the staging area
  • Run the command "git add",add a file to the staging area.

STEP 4:

  • Create a Commit
  • Run the command "git commit -m "the message"

Flags that can be used with Git Commit.

If you observed, when running the command git commit -m "message". You would been wondering what does the -m reperesents.Well, it reperesents a Flag!!

What is a Flag?

A flag enhance commands to work efficiently and are added using a "-" after the command.

Git commit consists of three flags and which are listed bleow.

  1. -a flag: this command commits any files you’ve added with the git add command and also commits any files you’ve changed since then.
  2. -m flag: this command is used to commit message to your commit
  3. -am flag: This command is a combination of the a and m flags and it is used to add and commit a message for the file in one command
